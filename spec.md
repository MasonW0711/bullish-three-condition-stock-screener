# 台股篩選器規格書（v2：多空雙向）

## 1. 目標

提供一套以 Streamlit 為介面的台股篩選系統，僅供研究參考使用。

本工具用來找出符合下列任一**訊號路徑**的股票，並依**方向（做多 / 做空）**分流呈現：

- **做多**：價格站回某條基準線之上（守在線上）。
- **做空**：價格被壓在某條基準線之下（壓不過線）。

方向**不由紅線 / 黑線決定**，而由「突破 / 跌破方向」與「回測時收在線上 / 線下」共同決定。紅線與黑線皆為突破基準線，僅產生來源不同（見 §3.3），兩者皆可同時出現在做多與做空結果中。

本工具不構成投資建議。

## 2. 產品範圍

### 2.1 本工具的四條訊號路徑

| 路徑 | 方向 | 觸發 | 回測判定 |
|------|------|------|----------|
| P1 突破回測 | 做多 | 向上突破紅線或黑線（§3.4a） | 守在線上（§3.5a） |
| P2 新線回測 | 做多 | 新紅線或黑線出現（§3.4c），不需突破 | 守在線上（§3.5a） |
| P3 跌破回測 | 做空 | 向下跌破紅線或黑線（§3.4b） | 壓在線下（§3.5b） |
| P4 新線回測 | 做空 | 新紅線或黑線出現（§3.4c），不需突破 | 壓在線下（§3.5b） |

> 註：本工具並無任何「評分（scoring）」機制；各路徑為「逐層過濾」，不是加權計分。一根 K 棒可同時符合多條路徑，但會依方向分流到不同輸出。

### 2.2 包含

- 台灣上市（`.TW`）與上櫃（`.TWO`）普通股
- 透過 yfinance 取得日線 OHLCV
- 由日線重取樣（resample）出週線 / 月線
- 大紅攻 / 大黑攻訊號偵測
- 由攻擊成功產生紅線 / 黑線
- 向上突破與向下跌破紅線 / 黑線
- 突破回測（做多守住）與跌破回測（做空壓回）
- 新線出現後窗格內的回測（不需突破，多空雙向）
- 選用的法人買賣超過濾
- Plotly K 線圖
- Excel 匯出（多空分工作表）

### 2.3 排除

- 下單
- 券商整合
- 回測引擎
- 投資組合管理
- 分鐘線（盤中資料）
- 突破緩衝、回測容差、量能比率、均量過濾等進階參數

## 3. 核心領域規則

### 3.1 攻擊方向

- `Open > prev_close` 只可能是「大紅攻」。
- `Open < prev_close` 只可能是「大黑攻」。
- 攻擊失敗不會轉成相反方向。

### 3.2 攻擊成敗

- 大紅攻成功：`Open > prev_close` 且 `Close > prev_close`
- 大紅攻失敗：`Open > prev_close` 且 `Close < prev_close`
- 大黑攻成功：`Open < prev_close` 且 `Close < prev_close`
- 大黑攻失敗：`Open < prev_close` 且 `Close > prev_close`

### 3.3 紅線 / 黑線

- 僅在「大紅攻成功」為真時，`red_line_raw = prev_close`。
- 僅在「大黑攻成功」為真時，`black_line_raw = prev_close`。
- `red_line` 與 `black_line` 各自在每一個 `StockCode` 內向前填補（forward-fill）。
- 「新線出現」定義：當根的 `red_line_raw`（或 `black_line_raw`）為非空，即視為該根產生一條新線（§3.4c 使用）。

### 3.4 突破 / 跌破 / 新線

線價泛指當前對應的 `red_line` 或 `black_line`（已 forward-fill）。

**a) 向上突破（做多觸發）**

- 向上突破紅線：前一根 `Close <= 前一根 red_line`，且當根 `Close > 當根 red_line`。
- 向上突破黑線：前一根 `Close <= 前一根 black_line`，且當根 `Close > 當根 black_line`。

**b) 向下跌破（做空觸發，本版新增）**

- 向下跌破紅線：前一根 `Close >= 前一根 red_line`，且當根 `Close < 當根 red_line`。
- 向下跌破黑線：前一根 `Close >= 前一根 black_line`，且當根 `Close < 當根 black_line`。

**c) 新線出現（不需突破，做多與做空共用觸發；本版新增）**

- 當根產生新紅線或新黑線（§3.3 定義）即記錄該線價與出現日期。
- 該線於出現後的 `new_line_window` 個交易日內，作為 P2 / P4 的回測基準線（見 §3.5c）。

**衝突優先序（顯示用）**

- 同一根同時向上突破紅線與黑線：顯示上以黑線優先。
- 同一根同時向下跌破紅線與黑線：顯示上以紅線優先。
- 突破與跌破不可能在同一根對同一條線同時成立（前一根 Close 不能同時 `<=` 與 `>=` 線價，除非等於；等於時歸類為「突破」不歸「跌破」）。

### 3.5 回測判定

**a) 做多守住（P1 / P2）**

- 有效：`Low <= 回測線價` 且 `Close >= 回測線價`。
- 若 `Close < 回測線價`，不算有效。

**b) 做空壓回（P3 / P4，本版新增）**

- 有效：`High >= 回測線價` 且 `Close <= 回測線價`。
- 若 `Close > 回測線價`，不算有效。

**c) 回測基準線來源**

- P1 / P3：以「最新被突破 / 跌破的線」在每檔股票內向前填補，作為回測基準線。
- P2 / P4：以「最新出現的新線」為基準線，且僅在新線出現後 `new_line_window` 個交易日內有效；超過窗格則該基準線失效，不再用於 P2 / P4。

### 3.6 時間框架順序

固定流程：

1. 下載日線資料
2. 重取樣為所選週期
3. 計算 `prev_close`
4. 計算攻擊訊號
5. 產生紅線 / 黑線（含新線出現標記）
6. 偵測向上突破與向下跌破
7. 偵測回測（做多守住、做空壓回）
8. 套用回看窗格、量能、以及選用的法人過濾
9. 依方向分流為做多 / 做空兩組輸出

訊號絕不可在日線上計算後再去重取樣。

### 3.7 法人連續日規則

`investor_consecutive_days = N`（可設定參數，預設 3）。

對於某根 K 棒日期：

- 外資連買條件成立，僅當最近 N 個交易日的 `foreign_net > 0` 全部成立。
- 投信連買條件成立，僅當最近 N 個交易日的 `trust_net > 0` 全部成立。
- 外資連賣條件成立，僅當最近 N 個交易日的 `foreign_net < 0` 全部成立。
- 投信連賣條件成立，僅當最近 N 個交易日的 `trust_net < 0` 全部成立。

法人資料以「該 K 棒日期當天或之前、最近一個有資料的法人日期」對應到每根 K 棒；旗標不得延伸到「最後一個有法人資料的日期」之後。

**多空法人過濾建議對應（選用）**

- 做多結果：可選用「外資 / 投信連買」過濾。
- 做空結果：可選用「外資 / 投信連賣」過濾。

### 3.8 方向分流（本版新增）

- 每根符合任一路徑的 K 棒標記 `direction ∈ {Long, Short}`。
- P1 / P2 → `Long`；P3 / P4 → `Short`。
- 同一根若同時符合做多與做空路徑（例如新線出現後同時滿足守住與壓回，極少見），分別輸出為兩列，各帶其 `direction` 與 `signal_type`。
- `signal_type ∈ {P1_BreakUp_Hold, P2_NewLine_Hold, P3_BreakDown_Reject, P4_NewLine_Reject}`。

## 4. 輸出契約

### 4.1 符合訊號

`final_signal == True` 的列，排序方式：

1. `Date` 由新到舊
2. `StockCode` 由小到大

並依 `direction` 拆為做多、做空兩組。

### 4.2 最新摘要

每檔股票、每個方向各一列，顯示回看窗格內「最新一筆有效訊號」（含 `signal_type`）。

### 4.3 Excel 匯出

固定工作表：

- `All_Data`
- `Long_Signals`（做多：P1 + P2）
- `Short_Signals`（做空：P3 + P4）
- `Latest_Summary_Long`
- `Latest_Summary_Short`
- `Failed_Downloads`（工作表名稱「下載失敗清單」；含「失敗股票代號」欄，以及「診斷訊息」批次層級的網路 / 來源錯誤診斷欄）
- `Parameter_Settings`

### 4.4 UI 分頁

Streamlit 以分頁（`st.tabs`）分開呈現「做多」與「做空」結果與各自 K 線圖。

## 5. 穩定不變式

- 週線使用該週「實際的最後一個交易日」。
- 月線使用該月「實際的最後一個交易日」。
- 最後交易日以 yfinance 實際回傳的最後一根日線為準，不補假日、不對齊到自然週期結尾。
- 任何 shift、forward-fill、rolling、resample 都必須以股票（`StockCode` / `BaseCode`）分組。
- UI 標籤、內部參數、Excel 參數表必須描述相同的行為。
- 做多回測判定一律用 `Close >= 線價`，做空一律用 `Close <= 線價`，不得混用。
- `new_line_window` 為可調參數（預設 5），其值必須同時反映在 UI、內部邏輯與 `Parameter_Settings`。

### 5.1 外部資料錯誤策略

- **結構性錯誤**（欄位數 / 表格格式與預期不符，例如股票清單表格欄數不對）→ **明確報錯**並指出來源，不可靜默吞掉。
- **暫時性錯誤**（單一股票下載失敗、單日法人 API 連線逾時 / 中斷）→ **隔離該筆、記錄原因、繼續執行**，不可中斷整批；失敗原因需可被使用者看到（`st.warning` 與 Excel `Failed_Downloads`），不可讓過濾條件靜默全部失效。
